05 / Notification center Persistent inbox opened from the bell icon. Tabs filter by type. Groups by time.

Anatomy

Every notification in Obsidian has the same four parts — this is how routing decides where it appears.

Category
fill · cancel · alert · deposit · news · kyc drives icon + routing
Severity
info · success · warn · error drives accent color
Persistence
transient · persistent transient → toast; both → center
Actionable
yes · no actionable gets primary CTAs

Routing rules

Fills and cancels → toast (4s) + center.
Price alerts → toast (4s) + sound + center.
Margin warnings → persistent banner + center + optional email.
News (high impact) → toast if user follows symbol; otherwise center-only.
KYC / compliance → modal the first time, banner thereafter.
Margin call → blocking modal + sound + SMS + email.
