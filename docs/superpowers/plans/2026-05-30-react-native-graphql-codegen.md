# React Native Mobile App — GraphQL Codegen Integration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire up `apps/mobile` (Expo) to consume the same enterprise GraphQL codegen pipeline as `apps/web` — shared types, shared hooks, same schema — so mobile gets typed data fetching "for free" when the backend team adds new fields.

**Architecture:** Three-layer shared schema approach:

1. `libs/shared/graphql-schema/` — versioned Git-tracked package holding generated `graphql.ts` (types) + `hooks.ts` (Apollo hooks). Both web and mobile `import from '@obsidian/graphql-schema'` — never generate into app dirs.
2. Codegen config at `apps/web/codegen.ts` writes to `libs/shared/graphql-schema/generated/`. A new `apps/mobile/codegen.ts` writes to the same destination.
3. `apps/mobile` (Expo) uses `@apollo/client` with React Native `HttpLink` + `InMemoryCache`, importing hooks from the shared package.

**Tech Stack:** Expo SDK 53 · React Native · Apollo Client 3.x · graphql-codegen · Nx 21 · TypeScript strict

    },
    // HOOKS — same shared package (single source of truth)
    '../libs/shared/graphql-schema/generated/hooks.ts': {
      plugins: ['typescript-operations', 'typescript-react-apollo'],
      documents: [
        '../apps/web/gql/operations/**/*.gql',

### Task 6: Create Apollo Client for React Native

**Files:**

- [ ] **Step 1: Create Apollo Client configuration**

```typescript
/**
 * File:        apps/mobile/src/lib/apollo-client.ts
 * Module:      Mobile · Apollo Client
 * Purpose:     Apollo Client instance configured for React Native.
 *              Uses HTTP link (no SSR) with InMemoryCache.
 *
 * Design decisions:
 *   - No SSR (unlike web's SchemaLink approach) — RN is always client-side
 *   - cache-and-network policy for fresh data without losing offline capability
 *   - persistCache disabled for now — can add expo-secure-store later
 *
 * Exports:
 *   - apolloClient — ApolloClient instance (singleton)
 *
 * Depends on:
 *   - @apollo/client — ApolloClient, InMemoryCache, HttpLink
 *
 * Side-effects:
 *   - Creates Apollo Client instance (no network calls until queries execute)
 *
 * Author:      BharatERP
 * Last-updated: YYYY-MM-DD
 */

import { ApolloClient, InMemoryCache, HttpLink, from } from '@apollo/client';

// The backend GraphQL endpoint — for local dev use your LAN IP or ngrok
// In production, this should come from environment variable EXPO_PUBLIC_GRAPHQL_URL
const GRAPHQL_ENDPOINT = process.env.EXPO_PUBLIC_GRAPHQL_URL ?? 'http://10.0.2.2:3000/graphql';

// HTTP link for queries and mutations
const httpLink = new HttpLink({
  uri: GRAPHQL_ENDPOINT,
  fetchOptions: {
    // Timeout for slow connections
    timeout: 15000,
  },
});

// Apollo Client instance — singleton pattern via module-level export
export const apolloClient = new ApolloClient({
  link: from([httpLink]),
  cache: new InMemoryCache({
    // Default cache config — merge policy for list updates
    typePolicies: {
      Query: {
        fields: {
          // pagination merge strategy — append to existing list
          instruments: {
            merge(_existing, incoming) {
              return incoming;
            },
          },
          orders: {
            merge(_existing, incoming) {
              return incoming;
            },
          },
          positions: {
            merge(_existing, incoming) {
              return incoming;
            },
          },
        },
      },
    },
  }),
  defaultOptions: {
    watchQuery: {
      fetchPolicy: 'cache-and-network',
      errorPolicy: 'all',
    },
    query: {
      fetchPolicy: 'cache-first',
      errorPolicy: 'all',
    },
    mutate: {
      errorPolicy: 'all',
    },
  },
});
```
